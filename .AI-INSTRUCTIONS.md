## 7. STANDARDS DU PROJET (Sprint 0) - OBLIGATOIRES

    ⚠️ TOUS LES NOUVEAUX MODULES DOIVENT UTILISER CES COMPOSANTS :

    ### 7.1 Logger Avancé (OBLIGATOIRE)
    ```python
    # ✅ TOUJOURS utiliser :
    from indexao.logger import get_logger
    logger = get_logger(__name__)

    # ❌ NE JAMAIS utiliser :
    import logging
    logger = logging.getLogger(__name__)
    ```

    **Fonctionnalités disponibles** :
    - Logs structurés avec contexte (logger.info/debug/error/warning)
    - Timer de performance : `with logger.timer("operation"):`
    - Métriques automatiques : `logger.metric("name", value)`
    - Trace level pour debug détaillé : `logger.trace(...)`
    - Configuration via TOML et variables d'environnement

    ### 7.2 Path Manager (OBLIGATOIRE pour I/O fichiers)
    ```python
    # ✅ TOUJOURS utiliser pour accès fichiers :
    from indexao.paths import get_path_adapter, FileInfo

    adapter = get_path_adapter("file:///data/docs")
    files = adapter.list_dir()
    content = adapter.read_file("document.txt")
    with adapter.open_stream("large.pdf") as stream:
        process(stream)

    # ❌ NE PAS utiliser pathlib.Path directement SAUF :
    # - Manipulation de chemins uniquement (sans I/O)
    # - Chemins internes/temporaires
    # - Configuration initiale
    ```

    **Avantages du Path Manager** :
    - Support multi-protocoles (file://, smb://, s3:// futures)
    - Cache des résolutions de chemins (performances)
    - Retry automatique sur erreurs transitoires
    - Métadonnées riches (FileInfo avec MIME type, permissions)
    - Thread-safe et production-ready
    - Logging intégré des opérations I/O

    ### 7.3 Quand utiliser quoi ?

    **Logger** : ✅ TOUJOURS dans TOUS les modules
    **Path Manager** : ✅ Pour toute opération de lecture/écriture de fichiers
    **pathlib.Path** : ✅ Uniquement pour manipulation de chemins (joinpath, parents, etc.)

    ### 7.4 Pattern d'import standard
